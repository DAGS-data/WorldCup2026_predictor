# V2 Prediction Engine — Plan

**Date:** 2026-07-04  
**Goal:** Replace Poisson draw-aware model with XGBoost binary classifier (advancement probability), incorporating tournament momentum, performance vs expectations, and external signals.

---

## 1. Problem Statement

### Current limitations
- Poisson models W/D/L independently — useless for knockout (no draws)
- No **tournament momentum**: Cape Verde (rank ~30) almost beat Argentina but model only sees Elo difference
- Flat treatment of all matches — beating Curaçao ≠ beating Germany
- No external data (news, injuries)

### V2 target

| Aspect | V1 (Poisson) | V2 (XGBoost) |
|--------|-------------|---------------|
| Output | W/D/L probs | P(Team A advances) — binary |
| Goal model | Independent Poisson | Bivariate correlation via features |
| Momentum | None | Weighted recent form × opponent quality |
| overperformance | None | xG overperformance, Elo delta tracking |
| Data | Static JSON | ESPN API + cron updates |
| Method | Statistical | ML ensemble (XGBoost + calibrated) |

## 2. Model Selection: Literature Review

### Why not Poisson?

**Dixon & Coles (1997)** showed independent Poisson **underestimates** low-scoring draws (0-0, 1-0) and overestimates rare high scores. Adds dependence parameter ρ for P(0,0), P(1,0), P(0,1), P(1,1).

### Academic SOTA

| Paper | Approach | Key finding |
|-------|----------|-------------|
| **Hubáček et al. (2019)** | LSTM + CNN on match sequences | DL on temporal data beats static Elo |
| **Berrar et al. (2019)** | XGBoost + 50+ engineered features | RPS 0.189, best non-DL approach |
| **Groll et al. (2019)** | Hybrid LASSO + RF + Poisson | Won 2018 WC prediction challenge |
| **FiveThirtyEight SPI** | Bayesian hierarchical + weighted Poisson | Tournament-specific match importance |
| **Beal et al. (2021)** | Graph Neural Networks | Teams as nodes, matches as edges |

### Chosen approach: XGBoost binary classifier

**Rationale:**
- ~100 training matches → XGBoost excels at this scale (NN would overfit)
- Groll et al. showed XGBoost + engineered features beats pure statistical models
- Binary target (advance/wins) directly maps to knockout requirements
- Built-in feature importance → explainable predictions
- Easy to update daily as new matches complete

## 3. Feature Engineering (50+ features)

### 3.1 Core (from Elo/rankings)
- `elo_diff`, `fifa_rank_diff`, `squad_value_ratio`, `host_advantage`, `confederation`

### 3.2 Tournament momentum (NEW — the "Cape Verde problem")
```python
# Performance vs expectation
# For each match: overperformance = actual_result - expected_result
# Exponentially weighted (λ=0.85 per match backward)

# Close-loss bonus  
# Lost by 1 goal to team with Elo >= +100 → score as 0.7 (not 0.0)

# Momentum score = Σ (result_weight × opponent_elo × recency_decay × match_importance)
# match_importance: group=1.0, R32=2.0, R16=3.0, QF=4.0

# Goal-scoring consistency: std(goals scored, last 5 matches)
# Defensive solidity: avg(goals conceded, last 3)
```

### 3.3 Tournament-specific
- `rest_days`, `travel_distance_km`, `group_position` (1st/2nd/3rd), `goal_difference_cumulative`, `clean_sheets`, `comeback_wins`, `extra_time_minutes`

### 3.4 Historical
- `h2h_wins_last_5`, `confederation_vs_confederation`, `wc_knockout_appearances`

### 3.5 External data (future)
- News sentiment via Google News RSS → NLP
- Injury reports
- Weather (temperature, humidity) — ESPN has this!

## 4. Implementation Plan

### Phase 1: Feature pipeline (`backend/feature_engineering.py` — NEW)
- Pull ESPN match data, compute all 50+ features per match
- Label: `advance=1` if team won (after ET/pens if applicable)
- Time-based train/test split

### Phase 2: Model training (`backend/models/xgboost_predictor.py` — NEW)
- XGBoost binary classifier (`objective='binary:logistic'`)
- Bayesian hyperparameter tuning (Optuna)
- Probability calibration (Platt scaling)
- Feature importance + SHAP explanations
- Save model to `backend/models/xgb_v1.json`

### Phase 3: API (`backend/main.py` — MODIFY)
- `/api/predict/v2?team1_id=X&team2_id=Y` → binary advancement probability
- `/api/predict` keeps Poisson for comparison

### Phase 4: Frontend
- Replace W/D/L display with "Advancement Probability" bar
- Remove draw column

## 7. Risks

| Risk | Mitigation |
|------|-----------|
| Overfitting (few matches) | L1/L2 reg, shallow trees, early stopping |
| Cold start (<5 matches) | Blend with Elo baseline |
| Data leakage | Strict time-based split; all features from pre-match data |
| ESPN rate limits | Cache + cron daily at 4 AM |
